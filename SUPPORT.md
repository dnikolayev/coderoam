# Support

`chat-bridge` is an early local-first project. Support is best-effort.

## Before Asking For Help

Run:

```sh
chat-bridge doctor
chat-bridge health
chat-bridge active status
chat-bridge logs tail
```

Include the command output with phone numbers, chat IDs, message bodies, QR
payloads, pairing codes, and session paths redacted.

## Good Support Questions Include

- Operating system and architecture.
- `chat-bridge` version or commit.
- Install method.
- Transport mode and login method.
- Whether a dedicated WhatsApp test account is used.
- The smallest reproduction steps.

## Unsupported Uses

The project does not provide support for:

- Bulk messaging.
- Marketing automation.
- Scraping contacts or group participants.
- Anti-ban behavior or platform evasion.
- Running hosted consumer-account automation for third parties.
- Using someone else's WhatsApp account.

## Security Issues

Do not open public issues for vulnerabilities. See `SECURITY.md`.
